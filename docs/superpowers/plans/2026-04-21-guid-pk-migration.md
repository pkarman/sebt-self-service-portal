# Guid Primary Key Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Spec:** [2026-04-21-guid-pk-migration-design.md](../specs/2026-04-21-guid-pk-migration-design.md)

**Goal:** Convert the `int` sequential primary keys on `Users`, `UserOptIns`, and `DocVerificationChallenges` to app-generated UUIDv7 (`Guid`). `DocVerificationChallenge.PublicId` is preserved as a separate column — it's an external contract boundary, not a storage optimization.

**Architecture:** Drop-and-recreate migration (tech-lead-confirmed no data to preserve in any environment). C# `Guid` types throughout; UUIDv7 generated in repository `CreateAsync`/`GetOrCreate*` paths via `Guid.CreateVersion7()` when `entity.Id == Guid.Empty`. JWT `sub` claim transparently carries the Guid string via the existing `user.Id.ToString()` call. Frontend and state-connector repos require **zero changes**.

**Tech Stack:** .NET 10, EF Core (SQL Server `uniqueidentifier`), xUnit + NSubstitute + Bogus + Testcontainers.
